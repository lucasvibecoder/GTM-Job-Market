## Summary

**Profound** helps companies understand and control their AI presence — positioning itself as a long-term AI technology partner. This **GTM Engineer** role owns the full go-to-market infrastructure: scoping problems across Sales, CS, Marketing, and Ops, then building agentic automation systems that give every team leverage. Heavy emphasis on manufacturing tools (not operating them) and thinking architecturally about data flow.

## Responsibilities

- Scope high-impact problems autonomously by shadowing Sales, CS, Marketing, and Operations teams to identify efficiency gaps
- Build scalable automation systems by designing and deploying agentic workflows using n8n, Claude API, MCP servers, and AI-native platforms
- Manufacture tools rather than operate them — create systems others can run independently
- Own the full technical stack across data pipelines, API integrations, schema design, and automation platforms
- Ship tools that people actually use — focus on enablement and adoption, not just deployment
- Think architecturally about data — structure schemas that power intelligent automation
- Experiment relentlessly with new platforms, APIs, and AI-native tools

## Requirements

### Must-Have
- High-agency builder who scopes problems, designs solutions, and ships without hand-holding
- Technically fluent in Python or JavaScript
- Comfortable building in n8n, integrating APIs, and understanding data architecture
- AI-native thinker who defaults to automation and experiments with agents, MCP, and workflow frameworks
- Systems thinker who architects solutions that create compounding value

### Nice-to-Have
- Experience with GTM workflows (sales, CS, marketing ops)
- Familiarity with Clay, PhantomBuster, Apify, Dust, or similar automation platforms
- Experience building with LLM APIs, vector databases, or agentic workflows
- Comfortable operating in ambiguity and structuring own roadmap

## Key Concepts

- **MCP Servers (Model Context Protocol)**: A protocol that lets AI models connect to external tools and data sources, enabling agents to take actions across systems rather than just generating text.
- **Agentic Workflows**: Automation pipelines where AI agents make decisions and take actions autonomously — beyond simple if/then logic — often chaining multiple LLM calls with tool use.
- **Schema Design for Automation**: Structuring data models so that information flows cleanly between systems, enabling downstream automation to operate reliably without manual cleanup.
