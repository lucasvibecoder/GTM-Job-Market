## Summary

**Grafana Labs** is the open-source observability powerhouse behind Grafana (20M+ users), powering monitoring at 3,000+ companies including Bloomberg, JPMorgan, and eBay. This **Staff GTM Engineer (AI & Automation)** builds next-generation multi-agent AI systems and workflow automation across Marketing, RevOps, and SDR teams. 60-70% coding, 30-40% operational enablement. Key strategic project: AI chatbot routing with human-in-the-loop orchestration. The deepest AI/agent engineering requirements in the vault — demands production LLM experience, multi-agent architecture, and systematic evaluation practices. Remote, US-based.

## Responsibilities

- Build and operate reliable GTM automation across custom code and orchestration tools (n8n/Workato/Make) with CI/CD, testing, and deployment practices
- Implement modular, scalable, multi-agent AI systems that operate 24/7 and integrate with marketing platforms (Customer.io, Marketo, Salesforce, BigQuery)
- Own technical direction for GTM automation: data models, API contracts, shared libraries, reference architectures
- Review code, write tests, and maintain production reliability standards
- Partner with RevOps, Demand Gen, Regional Marketing, Events Marketing, and SDRs to solve workflow inefficiencies through agent-based solutions
- Create reusable workflow templates, playbooks, and docs for team self-service
- Align AI chatbot systems across Marketing, SDR, and Customer Success for unified prospect routing
- Build intelligent routing logic for human-in-the-loop prospect interactions
- Instrument end-to-end funnel metrics (handoff time, qualification accuracy, meeting set rate, pipeline influence)
- Define SLAs, failure modes, and fallback pathways for AI systems
- Implement company-wide standards for AI prompt engineering, version control, testing, observability, and impact measurement
- Create internal training materials and scalable frameworks for AI tools

## Requirements

- 7+ years building production systems and integrations (software engineering, data engineering, business systems, or GTM engineering)
- 2+ years hands-on applying LLMs/AI to production workflows — not just prototypes
- Proven experience delivering AI-enabled systems with measurable business outcomes
- Strong in Python and JavaScript/Node.js with Git-based development, code review, and testing discipline
- Deep familiarity with Google Cloud Platform, BigQuery, and Cloud Functions
- LLM frameworks and patterns: prompt engineering, RAG, function calling/tool use, agent orchestration, structured output parsing, evaluation
- Fluent with AI-assisted dev tools (GitHub Copilot, Cursor, Claude Code)

- LLM failure modes and production mitigations: confidence thresholds, fallback logic, human escalation, cost/latency management
- Systematic LLM evaluation: golden datasets, regression testing, A/B testing prompts, monitoring for drift
- Conversational AI, chatbot development, and routing logic
- Integrating with Salesforce and at least one marketing automation platform (Customer.io / HubSpot / Marketo)

## Key Concepts

- **Multi-Agent Architecture**: A system design where multiple specialized AI agents handle different tasks (routing, classification, response generation) and coordinate through orchestration patterns — sequential chains, router/dispatcher, or parallel fan-out — rather than one monolithic model doing everything.
- **Human-in-the-Loop Orchestration**: AI systems that know when to escalate to humans — using confidence thresholds, intent classification, and fallback logic to route interactions between autonomous agents and human operators based on complexity and risk.
- **LLM Evaluation (Evals)**: Systematic testing of AI system quality using golden datasets, regression tests, prompt A/B tests, and drift monitoring — ensuring production AI doesn't silently degrade over time.
