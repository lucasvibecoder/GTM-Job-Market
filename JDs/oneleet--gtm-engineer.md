## Summary
Oneleet is a security and compliance platform ($33M Series A) that helps companies build, manage, and monitor cybersecurity programs for SOC 2 and ISO 27001. The GTM Engineer is a founding role — single owner of the entire GTM tech stack, data infrastructure, and attribution model. You architect how data flows from first ad click to closed-won deal, connecting Clay, HubSpot, Dreamdata, Google Tag Manager, Google Ads, Looker Studio, and CrazyEgg. Equal parts builder and strategist: designing experiments, connecting platforms via APIs, building attribution models, and translating data into revenue insights.

## Responsibilities
- Own every tool in the GTM stack — decide how they connect, what data flows where, and why
- Evaluate, implement, configure, and maintain Clay, HubSpot (CRM, Marketing Hub, Operations Hub), Dreamdata, Google Tag Manager, Google Ads, Looker Studio, and CrazyEgg
- Build integrations using APIs, webhooks, middleware, or custom code when native connections don't exist
- Design the master data model ensuring a single source of truth across all systems
- Design and implement the entire UTM strategy and multi-touch attribution model from scratch
- Configure Dreamdata for B2B revenue attribution and set up Google Tag Manager with proper event tracking and data layer implementation
- Build Looker Studio dashboards showing executives where pipeline and revenue originate
- Create tracking infrastructure (CrazyEgg heatmaps, GA4 events, conversion tracking) capturing every meaningful interaction
- Own data quality, enrichment workflows, lead scoring models, and the journey from anonymous visitor to closed deal
- Design and run experiments across the funnel — A/B tests on landing pages, outbound sequence variations, lead routing optimizations, scoring model adjustments
- Build experimentation infrastructure: proper tracking, clean control groups, statistically valid measurement
- Use Clay to build enrichment and prospecting workflows, test new data sources, and scale what works into production automations
- Deliver enriched lead data, buying signals, and account intelligence for sales conversations
- Build dashboards and reports for executives showing which channels, campaigns, and motions drive pipeline and revenue
- Sit in revenue meetings, explain what data shows, and recommend where to invest next
- Create documentation so the infrastructure scales beyond you

## Key Concepts
- **Multi-touch attribution model**: A framework for assigning credit to multiple marketing touchpoints along the buyer journey, rather than just first-touch or last-touch. Oneleet wants this built from scratch using Dreamdata and GTM event tracking.
- **Master data model**: A single authoritative schema defining how data entities (leads, accounts, opportunities, campaigns) relate across all connected systems. Ensures every tool in the stack agrees on what a "lead" or "pipeline" means.
- **Generative Engine Optimization (GEO)**: Not explicitly mentioned here, but the experimentation and attribution infrastructure this role builds is the measurement layer that tells the GTM team which motions actually work.
