## Summary

xpander.ai builds the runtime and control plane for production AI agents — deploy, operate, and scale agents across cloud and self-hosted environments with reliability, security, and observability. This is the first GTM Engineer hire. A builder role with customer accountability: take the technical GTM motion from zero to repeatable by running deep technical discovery, building and deploying real agent systems with customers, writing production code, and creating the repeatable GTM playbook.

## Responsibilities

- Take the technical GTM motion from zero to repeatable
- Run deep technical discovery and architecture conversations with customers
- Build and deploy real agent systems with customers — hands-on evaluations, POCs, and early production deployments
- Write code, configs, and examples that customers actually use
- Debug agent behavior, infra issues, and integrations
- Deliver demos that show production reality, not slides
- Turn repeated friction into concrete product and docs changes
- Maintain tight feedback loops from customers into product and roadmap
- Build clear signal on ICP, use cases, and where xpander wins or loses

## Requirements

### Must-Have

- Strong engineer first — can ship and debug real systems
- Comfortable working directly with customers and founders
- Experience with backend platforms, infra, APIs, and cloud systems
- Clear, direct communicator — no hand-waving
- Bias toward action and ownership in ambiguous environments

### Nice-to-Have

- Experience with AI agents, LLM infrastructure, or developer platforms

## Key Concepts

- **AI Agent Runtime**: The infrastructure layer that executes AI agents in production — handling deployment, scaling, monitoring, and reliability. Different from building the agent itself; this is the platform the agent runs on.
- **Control Plane**: The management layer that coordinates and configures distributed systems — in xpander's case, orchestrating how AI agents are deployed, monitored, and scaled across environments. Borrowed from Kubernetes/cloud infrastructure terminology.
