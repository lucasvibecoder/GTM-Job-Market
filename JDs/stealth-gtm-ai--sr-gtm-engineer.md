## Responsibilities

- Build connected revenue engines that enable client teams to operate faster and more efficiently
- Craft and position customer ICP and TAM documentation
- Build outbound systems with email deliverability and effective copywriting
- Create Clay-centered deliverables: enrichment, segmentation, scoring, routing, research, triggers, and automation
- Automate key GTM system aspects: lead routing, data entry, follow-ups, sales processes
- Design and implement scalable, reliable Clay workflows
- Connect disparate systems to create seamless, unified data flow via Clay integrations (CRMs, sequencing tools, enrichment providers, data warehouses, webhooks)
- Cleanse customer CRM data, build data pipelines, and create reporting dashboards
- Leverage AI for predictive lead scoring, hyper-personalization at scale, and signal/intent aggregation
- Build dashboards tracking data integrity, match rates, enrichment accuracy, and workflow uptime
- Create reports identifying pipeline bottlenecks and improving key conversion metrics

## Key Concepts

- **Multi-Agent Workflows**: Chaining multiple AI agents together to handle complex GTM tasks end-to-end — e.g., one agent researches accounts, another scores them, another generates personalized outreach. Explicitly called out as a must-have skill here.
- **Clay-Centered Deliverables**: Using Clay as the orchestration hub for client engagements — enrichment tables, scoring models, routing logic, and automation triggers all built inside Clay and integrated outward to CRMs and sequencing tools.
- **System Orchestration via AI**: Using AI not just for content generation but to coordinate across systems — predictive scoring, signal aggregation, and automated workflows that replace manual RevOps processes.
