## Summary
NetRise is a cybersecurity platform helping security teams reduce software supply chain risk with visibility into what's running in their environments. The GTM Engineer owns outbound pipeline development — building the signal-to-workflow-to-message engine, executing multi-channel outreach (email, LinkedIn, phone), and generating qualified meetings. Blends high-output SDR execution with systems-building using AI martech tools.

## Responsibilities
- Build and run outbound prospecting motions that consistently create qualified first meetings
- Design and continuously improve the GTM "signal → workflow → message" engine
- Define signals that matter (persona fit + timing + trigger events)
- Set up alerts, routing, prioritization, and follow-up logic
- Operate and optimize tooling across the outbound stack (Clay, SalesLoft, HG Insights, Gong, enrichment tools, LinkedIn tooling)
- Create and maintain segmented prospect lists using firmographics, technographics, and buyer-role mapping
- Craft high-quality, personalized messaging that reflects real buyer value
- Run A/B tests on messaging, sequencing, and channel strategy
- Partner with Marketing on converting intent and inbound engagement into outbound follow-up
- Document repeatable playbooks (signal definitions, list builds, sequences, handoff rules)
- Bring a systems approach to pipeline generation: dashboards, metrics, and weekly iteration loops

## Requirements

### Must-Have
- 1-4+ years in SDR/BDR, outbound AE, RevOps/Growth, or GTM systems role (cybersecurity or technical B2B preferred)
- Proven ability to generate meetings via multi-channel outbound (email + LinkedIn + phone)
- Hands-on modern GTM tooling, especially LLM systems for personalized content (strong preference for Clay + sequencing platform like SalesLoft)
- Deep understanding of intent signals and triggers (tech stack changes, hiring, funding, job changes, compliance, incident-driven urgency)
- Strong written communication — concise, credible copy that security leaders respect
- Analytical mindset — track results, diagnose drop-offs, improve workflows independently
- Operational excellence — CRM and data hygiene tight, attribution clean
- Ability to engage technical personas (product security, TPRM, GRC, VM/IR) with curiosity and credibility

### Nice-to-Have
- Call recording/analysis tools (Gong)
- Enrichment vendors
- SQL or spreadsheets
- Lightweight automation (Zapier/Make, APIs)

## Key Concepts
- **Signal → workflow → message engine**: A structured system where buying signals (trigger events, intent data) feed into automated workflows that produce personalized outreach. The core architecture this role builds and iterates on.
- **Firmographic and technographic segmentation**: Building prospect lists based on company attributes (size, industry, funding) and technology stack data (what tools they use). Goes beyond basic "first-name personalization" to real buyer-value messaging.
