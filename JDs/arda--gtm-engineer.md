## Summary

Arda modernizes inventory and order management for manufacturers with scannable Kanban cards and AI-powered replenishment workflows. Just raised a $3.5M seed round. This GTM Engineer owns the full revenue engine — building systems across outbound, inbound, lifecycle, and expansion — while serving as the GTM Scrum Master running sprint-based experimentation. Works directly with leadership to scale to $10M ARR.

## Responsibilities

- Identify revenue bottlenecks across outbound, inbound, lifecycle, and expansion
- Design and ship workflows that turn good ideas into repeatable programs
- Own the GTM data foundation: definitions, hygiene, enrichment strategy, single source of truth
- Connect the GTM stack (CRM, sequencing, enrichment, product data, support data) so information flows without manual work
- Use AI, APIs, webhooks, automations, SQL, and lightweight scripting to run experiments and instrument GTM operations
- Create reliable infrastructure: alerts, monitoring, deduplication, QA checks, documentation
- Define hypotheses and success metrics for GTM plays (conversion rates, speed-to-lead, reply rates, meeting rates, CAC signals, pipeline quality)
- Build dashboards and reporting that leaders actually use to make decisions
- Run A/B tests and controlled rollouts
- Maintain the GTM backlog: prioritize, write clear tickets/specs, define "done"
- Run GTM sprint ceremonies: planning, standups, demos, retros
- Keep cross-functional execution moving (sales, marketing, CS, product)

## Requirements

### Must-Have

- Built and owned revenue, ops, or growth systems in a startup/high-ownership environment
- Systems and feedback loop thinking
- Hands-on with spreadsheets, SQL, APIs/webhooks, lightweight scripting (Python or JS)
- Experience with GTM tools (Clay, Unify) and AI tools
- Opinion about outreach sequences and channels
- Translate between sales/marketing goals and technical implementation without losing meaning
- Run a sprint process
- Measure outcomes, document decisions, and say "no" with clear rationale

### Nice-to-Have

- Designing experiments and interpreting noisy GTM data with intellectual honesty
- CRM architecture (HubSpot), lifecycle stage design, and attribution caveats
- Partnering directly with founders on ambiguous, high-impact work

## Key Concepts

- **GTM Scrum Master**: Applying agile sprint methodology to go-to-market — running 1-2 week sprints with backlogs, ceremonies, and measurable outcomes instead of treating GTM as a collection of activities. Unique hybrid of GTME + project management.
- **Compounding Workflows**: Automation systems that build on each other over time — e.g., enrichment feeding segmentation feeding personalized outbound. The goal is leverage that grows, not one-off fixes.
