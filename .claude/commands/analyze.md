# /analyze — Generate Analysis Artifacts

Regenerate all `Analysis/` and `Hubs/` files from `JDs/` data. Overwrites existing `Analysis/` and `Hubs/` files.

## Prerequisites
Read these files before starting:
- `Context/Archetypes.md` (archetype definitions)
- `Context/Tags.md` (tool normalization)
- `Context/Profile.md` (for preference alignment)
- All files in `JDs/` (market data — read all frontmatter)
- All files in `Dream/` (for archetype mapping and preference alignment only — **exclude from frequency counts**)

## Artifacts to Generate

### 1. Analysis/Tool_Frequency_Map.md
For each tool across all `JDs/` files:
- Count how many JDs mention it
- Calculate % of total JDs
- Assign tier: **Table-stakes** (60%+), **Differentiator** (25-59%), **Rare** (<25%)
- Sort by count descending

Format as table:
```
| Tool | Count | % of JDs | Tier |
```

Add a section at the bottom: **Lucas's Stack Coverage** — which table-stakes tools Lucas knows (from Profile.md) vs gaps.

### 2. Analysis/Role_Archetype_Clusters.md
Group JDs by primary archetype. For each cluster:
- List member JDs (company — title)
- Common patterns (shared tools, shared skill requirements, salary range)
- What distinguishes this cluster

Include Dream/ roles in archetype mapping but mark them with `[Dream]`.

### 3. Analysis/Skills_Tier_Map.md
For each skill across all `JDs/` files:
- Count total mentions
- Split by required vs preferred
- Calculate % of total JDs
- Assign tier: Table-stakes / Differentiator / Rare
- Sort by total count descending

Format as table:
```
| Skill | Required | Preferred | Total | % | Tier |
```

### 4. Analysis/Salary_Bands.md
Group salary data by:
- **Archetype** — range, median, mean for each archetype
- **Seniority** — Junior (0-2 YOE), Mid (2-4 YOE), Senior (5+ YOE)
- **Overall** — market range, median, mean

Note which JDs have OTE vs base salary. Flag JDs with no salary data.

Format as tables with ranges.

### 5. Analysis/Preference_Alignment.md
Rank all JDs (including Dream/) by preference fit:
- **Annotation sentiment**: Positive annotations ("sounds fun", "sounds solid") = boost. Negative ("nooo", "Meh") = penalize. Mixed = note both.
- **Preference signals from Profile.md**: Building > managing, autonomy, AI-forward, early-stage, remote-friendly, no dashboards.
- Score each JD qualitatively: Strong Fit / Good Fit / Mixed / Weak Fit

List top 5 best-fit roles with reasons.

### 6. Analysis/Fit_Gap_Analysis.md
Generated by `/gap` skill, not `/analyze`. Create a placeholder:
```
# Fit & Gap Analysis
Run `/gap` to generate this file.
```

### 7. Hubs/Archetypes/ and Hubs/Tools/
Wipe and regenerate `Hubs/` from scratch.
- 5 archetype hub pages from `Context/Archetypes.md` + JD frontmatter
- Tool hub pages for every tool appearing in 3+ JDs
- See CLAUDE.md §9 for templates and conventions

## Post-Generation
After writing all files, update `Context/Archetypes.md` if analysis reveals:
- A cluster that doesn't match any existing archetype
- An archetype with zero or one members (consider merging)
- New distinguishing patterns worth documenting

Print summary:
```
Analysis complete.
- X JDs analyzed (Y in Dream/ for archetype mapping only)
- [Archetype distribution summary]
- Top table-stakes tools: [top 3]
- Salary range: $X - $Y (median $Z)
```
